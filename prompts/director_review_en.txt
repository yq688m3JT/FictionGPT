## Review Requirements
Please review the text's consistency with the plan from the following dimensions:
1. **Narrative goal achievement**: Does the text achieve the planned narrative goal? Were key plot points omitted or severely deviated from?
2. **Character behavior consistency**: Are characters' words and actions consistent with their personality settings and current state?
3. **Foreshadowing execution**: Is the foreshadowing that was planned to be planted/recalled reflected in the text?
4. **Pacing and tone**: Are the text's pacing and emotional tone consistent with the plan?
5. **Continuity**: Are there contradictions or disconnects with previous chapters?

## Output JSON Schema (Strict compliance required)
```json
{{
  "needs_revision": <bool, whether rewriting is needed. Only true when severe deviations exist; minor differences are considered reasonable creative liberty by the writer>,
  "revision_notes": "<string, if revision needed: specify which paragraphs need what changes. If no revision needed: empty string>",
  "score": <int, 1-10 rating>,
  "deviations": ["<string, description of discovered deviations, list even if revision is not needed>"],
  "architecture_adjustments": "<string, if the text contains unexpected but reasonable developments, whether subsequent narrative architecture needs adjustment. Empty string if no adjustment needed>"
}}
```

## Review Principles
1. **Tolerate reasonable creativity**: The writer's creative liberties at the detail level (dialogue wording, environmental description, micro-expressions) do not count as deviations. Only deviations that affect narrative direction, character development direction, or foreshadowing system need to be flagged.
2. **Strict causal logic**: If the text introduces unplanned major events or character behavior logic breaks, it must be flagged as needing revision.
3. **Protect foreshadowing system**: If foreshadowing that was planned to be planted was omitted, it must be noted in revision comments.
4. **Scoring standard**: 8-10 = excellent execution, 6-7 = acceptable with minor flaws, below 5 = needs rewriting. needs_revision is only true at 5 or below.
5. Output only valid JSON, no explanatory text.
