## 你的任务
基于叙事解构中的预设规划和当前最新的故事状态，输出本章的最终执行规划。你可以在以下范围内微调：
- 根据前章实际内容调整场景细节和情感节奏
- 补充叙事解构中未预见到的合理细节
- 但**不得**改变本章的叙事目标、关键事件方向和伏笔计划

```json
{{
  "chapter_number": <int>,

  "narrative_goal": "<string，一句话概括本章在整体叙事弧中的作用>",
  "narrative_position": "<string，当前在整体故事弧的位置>",
  "key_scenes": [
    {{
      "scene_id": "<章节号-场景序号>",
      "location": "<string，具体场景地点>",
      "characters": ["<角色名>"],
      "conflict": "<string，本场景核心矛盾或张力>",
      "emotion_arc": "<string，情绪变化轨迹>",
      "pacing": "<string：急促 | 舒缓 | 紧绷 | 留白>"
    }}
  ],
  "foreshadowing": {{
    "plant": ["<新伏笔描述>"],
    "recall": ["<要回收的旧伏笔描述>"]
  }},
  "tone": "<string，本章整体叙事基调>",
  "word_count_target": <int，建议字数>
}}
```

## 规划原则
1. 本章规划必须与叙事解构保持一致，微调范围限于细节层面。
2. 角色行为必须符合其当前状态与性格。
3. 按叙事解构的伏笔计划执行：本章应埋设的伏笔不得遗漏，应回收的伏笔必须安排场景。
4. narrative_goal 必须说明本章对整体故事弧的具体贡献。
5. 只输出合法 JSON，不要输出任何解释性文字或 markdown 代码块标记。
