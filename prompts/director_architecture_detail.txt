## 输出 JSON Schema（严格遵守）
```json
{{
  "chapters_detailed": [
    {{
      "chapter_number": <int>,
      "chapter_title_hint": "<string，章节参考标题>",
      "narrative_goal": "<string，本章在整体叙事弧中的作用>",
      "narrative_position": "<string，铺垫期/矛盾激化/高潮前夕/高潮/余波>",
      "story_lines_advanced": ["<故事线id>"],
      "key_scenes": [
        {{
          "scene_id": "<章节号-场景序号>",
          "location": "<string，场景地点>",
          "characters": ["<角色名>"],
          "conflict": "<string，核心矛盾或张力>",
          "emotion_arc": "<string，情绪变化轨迹>",
          "pacing": "<string：急促 | 舒缓 | 紧绷 | 留白>"
        }}
      ],
      "foreshadowing": {{
        "plant": ["<新伏笔描述>"],
        "recall": ["<要回收的旧伏笔描述>"]
      }},
      "tone": "<string，本章整体叙事基调>",
      "word_count_target": <int，建议字数>
    }}
  ]
}}
```

## 注意
1. 严格按照骨架中每章的 narrative_goal 和 narrative_position 来展开细节，不要偏离整体规划。
2. 场景设计要服务于该章的叙事目标，每个场景都要有明确的冲突或情感推动力。
3. 伏笔的 plant/recall 必须与骨架中的 foreshadowing_plan 一致。
4. 只输出合法 JSON，不要输出任何解释性文字或 markdown 代码块标记。
